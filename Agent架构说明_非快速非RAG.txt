GeoAgent Agent 架构说明（排除快速模式与 RAG 模式）

生成时间：2026-06-09
项目模块：app
主要包名：com.geoagent.agent / com.geoagent.agent.v2 / com.geoagent.ui.chat

一、本文范围

本文只整理当前项目中“快速模式”和“RAG 模式”以外的 Agent 架构。

不展开说明的范围：
1. 快速模式：普通 Chat/快速问答链路，即未命中特定 Agent 时走 ChatRepository.streamChat 的普通对话。
2. RAG 模式：知识库检索模式及 V2 RAG Agent 的具体回答逻辑。

本文覆盖的 Agent：
1. Search Agent / 联网搜索
2. Research Agent / 研究分析
3. Schedule Agent / 日程安排
4. Task Agent / 任务管理
5. Email Agent / 邮件助手
6. PDF Agent / PDF 解析
7. File Agent / 文件管理
8. UnitConversionAgent / 单位换算本地 Agent
9. Document / Settings 两个 UI 导航类 Agent


二、总体架构

当前 Agent 架构分为两层：

1. 入口路由层
   文件：app/src/main/java/com/geoagent/agent/AgentRegistry.kt
   核心类：
   - IntentRouter
   - AgentMeta
   - RouteResult
   - SessionContextManager
   - BuiltinAgents

   职责：
   - 对用户输入做关键词、正则、semanticHints 打分。
   - 根据 confidence 判断 DIRECT / CONFIRM / FALLBACK。
   - 维护短期上下文 sessionContext，使连续请求可以继承上一次 Agent。
   - 将 V2 Agent 通过 BuiltinAgents.V2_BRIDGED_AGENTS 桥接到旧路由层。

2. V2 Runtime 编排层
   文件：
   - app/src/main/java/com/geoagent/agent/v2/V2AgentSystem.kt
   - app/src/main/java/com/geoagent/agent/v2/V2Runtime.kt

   核心类：
   - V2AgentRegistry
   - V2ToolRegistry
   - V2Orchestrator
   - V2RuntimeOrchestrator
   - V2RuntimeAgentExecutorRegistry
   - V2RuntimeGateway

   Pipeline 阶段：
   MASTER -> PLANNER -> ROUTER -> AGENTS -> REFLECTION -> JUDGE -> AGGREGATOR

   职责：
   - Master 接收用户请求。
   - Planner 将输入拆成一个或多个 V2PlanTask。
   - Router 为每个 task 选择 V2AgentId。
   - Agents 调用具体 Runtime Executor。
   - Reflection 检查缺失工具、空输出、缺少 artifact 等问题。
   - Judge 判断执行是否通过。
   - Aggregator 汇总最终 answer 和 runs。


三、V2 Tool Registry

文件：app/src/main/java/com/geoagent/agent/v2/V2AgentSystem.kt

工具枚举：
1. ONE_LLM
   - 类型：LLM_VIRTUAL
   - 用途：各虚拟 Agent 共享的单一 LLM 能力。

2. TAVILY_SEARCH
   - 类型：EXTERNAL_API
   - 用途：联网搜索。

3. ROOM_MEMORY
   - 类型：LOCAL_DATABASE

4. RETROFIT_NETWORK
   - 类型：EXTERNAL_API
   - 用途：网络请求能力。

5. FLOW_STREAMING
   - 类型：LOCAL_ANDROID
   - 用途：Kotlin Flow 事件流和内容流式输出。

6. TASK_MANAGER
   - 类型：LOCAL_DATABASE
   - 用途：任务保存、任务列表。

7. CALENDAR_CONTRACT
   - 类型：ANDROID_SYSTEM
   - 用途：Android CalendarContract 日历候选事件。

8. JAVA_MAIL_QQ_SMTP
   - 类型：EXTERNAL_API
   - 用途：QQ SMTP 邮件发送。

9. PDF_PARSER
   - 类型：LOCAL_ANDROID
   - 用途：PDF 文本解析。

10. FILE_ACCESS
    - 类型：LOCAL_ANDROID
    - 用途：本地文件/文档访问。

11. DOCUMENT_WRITER
    - 类型：LLM_VIRTUAL
    - 用途：结构化写作、改写、摘要等虚拟写作能力。


四、路由规则概览

文件：app/src/main/java/com/geoagent/agent/v2/V2AgentSystem.kt

V2AgentRegistry.route(input) 的流程：
1. normalize(input)
2. 遍历生产 Agent 列表 productionAgents()
3. 对每个 Agent 计算：
   - keywordHits
   - regexHits
   - priorityBoost
   - Email Agent 的 explicitEmailSend 特殊加权
4. 过滤 confidence > 0.35 的候选
5. 先按 confidence 降序，再按 priority 升序排序
6. 返回最高分候选

注意：
- Email Agent 对 “发给/发送给/发送至/发至/给 + 邮箱地址” 有显式高置信度识别。
- Search Agent 的“新闻/最新/搜索”容易命中，但在明确邮件发送场景中会被 Email Agent 抢占。
- File Agent 的“文件/上传/导入/保存/读取/查看/打开/删除 + 文件扩展名”用于命中文件处理。


五、UI 到 Runtime 的调用链

文件：app/src/main/java/com/geoagent/ui/chat/ChatManager.kt

调用链：
1. ChatActivity 收集用户输入，调用 ChatViewModel/ChatManager.sendMessage。
2. ChatManager 使用 IntentRouter.route 做入口路由。
3. UnitConversionAgent 若 DIRECT 命中，会走本地直接执行。
4. 其他 V2 Agent 若需要 runtime 回答，则调用 dispatchV2RuntimeMessage。
5. dispatchV2RuntimeMessage 创建 V2RuntimeRequest：
   - input：当前用户输入
   - imageBase64：可选图片
   - history：最近若干条非空聊天消息
6. 调用 V2RuntimeOrchestrator.orchestrate。
7. Runtime 通过 onEvent 更新状态文案，通过 onContent 流式写入 Agent 输出。
8. 执行完成后 V2OrchestrationResult 被转换成 assistant Message。

Loading 文案：
- Email Agent：邮件发送中...
- Search Agent：智能搜索中...
- RAG Agent：知识库检索中...
- 其他 V2 Agent：
  - MASTER / PLANNER / ROUTER：正在分析请求...
  - AGENTS：正在执行任务...
  - REFLECTION / JUDGE / AGGREGATOR：正在整理结果...


六、Agent 详情

1. Search Agent / 联网搜索

AgentId：
- V2AgentId.SEARCH
- wireName：search

触发关键词/正则：
- 搜索
- 联网
- tavily
- 最新
- 新闻
- Regex：(联网|搜索|查一下|最新|新闻)

优先级：
- priority = 10

依赖工具：
- ONE_LLM
- ROOM_MEMORY
- FLOW_STREAMING
- TAVILY_SEARCH
- RETROFIT_NETWORK

执行器：
- V2RuntimeSearchExecutor

执行流程：
1. gateway.search(task.input)
2. SearchUseCase 通过 Tavily 获取 SearchContext
3. 提取 sources，最多取 8 条作为 evidence
4. gateway.streamWithMemory 调用 One LLM 合成回答
5. 回答末尾拼接 Sources 列表
6. gateway.remember 写入 Room memory
7. 生成 search artifact：
   - sources
   - urls
   - titles

失败模式：
- Tavily 搜索失败 -> BLOCKED
- One LLM 合成失败 -> BLOCKED

说明：
- Search Agent 属于联网搜索 Agent，不属于“快速模式”。
- 如果用户打开“智能搜索”按钮，普通 Chat 链路也可能走搜索增强；本文此处描述的是 V2 Search Agent runtime。


2. Research Agent / 研究分析

AgentId：
- V2AgentId.RESEARCH
- wireName：research

触发关键词/正则：
- 研究
- 调研
- 综述
- 论文研究
- 资料整理
- 分析
- Regex：(研究|调研|综述|资料整理|分析)

优先级：
- priority = 12

依赖工具：
- ONE_LLM
- ROOM_MEMORY
- FLOW_STREAMING
- TAVILY_SEARCH
- RAG_RETRIEVAL
- DOCUMENT_WRITER

执行器：
- V2RuntimeResearchExecutor

执行流程：
1. gateway.search(task.input)，尝试获取联网资料。
2. gateway.retrieveRag(task.input, topK = 4)，尝试获取本地资料片段。
3. 将联网 Web evidence 和本地 Doc evidence 拼成研究证据。
4. gateway.streamWithMemory 调用 One LLM 生成研究分析。
5. gateway.remember 写入 research 记忆。
6. 生成 research artifact：
   - webSources
   - chunks
   - sourceUrls
   - documents

失败模式：
- search / RAG 可为空或失败，但 Research Agent 会尽量继续。
- One LLM 合成失败 -> BLOCKED。

说明：
- Research Agent 是综合型 Agent，它可以同时使用联网搜索和本地文档证据，但不等同于 RAG 模式。


3. Schedule Agent / 日程安排

AgentId：
- V2AgentId.SCHEDULE
- wireName：schedule

触发关键词/正则：
- 安排
- 排期
- 时间表
- 计划表
- schedule
- Regex：(安排|排期|时间表|schedule)

优先级：
- priority = 31

依赖工具：
- ONE_LLM
- ROOM_MEMORY
- FLOW_STREAMING
- CALENDAR_CONTRACT
- DOCUMENT_WRITER

执行器：
- V2RuntimeScheduleExecutor

执行流程：
1. parseV2TimeWindow 尝试解析时间。
   - 支持“几分钟后/几小时后/几天后”
   - 支持“今天/明天/后天/大后天”
   - 支持“周一/星期二/下周三”等
   - 支持“几点/HH:mm/上午/下午/晚上”等
2. gateway.streamWithMemory 调用 One LLM 生成时间块计划。
3. gateway.saveTask 保存为任务记录。
4. 如果解析出具体时间，则 gateway.prepareCalendarEvent 构造 Android 日历候选事件。
5. gateway.remember 写入 schedule 记忆。
6. 生成 schedule artifact：
   - taskId
   - calendar
   - begin
   - end
   - timezone

失败模式：
- One LLM 调用失败 -> BLOCKED
- 保存任务失败 -> BLOCKED
- 没有具体时间时仍可完成，但 followUps 会提示补充日期时间后可写入日历。

UI 后续动作：
- V2SystemActionMapper 可将 calendar/schedule/meeting/travel 类型 artifact 转为 OpenCalendarInsert。
- ChatActivity 收到 pendingSystemAction 后可打开系统日历插入界面。


4. Task Agent / 任务管理

AgentId：
- V2AgentId.TASK
- wireName：task

触发关键词/正则：
- 任务
- 待办
- todo
- 事项
- 清单
- Regex：(任务|待办|todo|清单)

优先级：
- priority = 32

依赖工具：
- ONE_LLM
- ROOM_MEMORY
- FLOW_STREAMING
- TASK_MANAGER
- ROOM_MEMORY

执行器：
- V2RuntimeTaskExecutor

执行流程分支：

分支 A：查看任务列表
1. isTaskListIntent 判断：
   - 有任务词：任务/待办/todo/清单/事项
   - 有查看词：查看/列出/显示/有哪些/当前/list/show
   - 没有创建词：创建/新增/添加/保存/安排/制定/生成
2. gateway.listTasks(status = "open", limit = 20)
3. 若为空，输出“当前没有未完成任务。”
4. 若非空，按行输出任务状态、标题、来源 Agent、优先级、id
5. 生成 task_list artifact：
   - count
   - status
   - ids

分支 B：创建/保存任务
1. gateway.streamWithMemory 调用 One LLM 拆解任务。
2. gateway.saveTask 保存任务。
3. gateway.remember 写入 task 记忆。
4. 生成 task artifact：
   - id
   - title

失败模式：
- 任务列表读取失败 -> BLOCKED
- One LLM 调用失败 -> BLOCKED
- 任务保存失败 -> BLOCKED


5. Email Agent / 邮件助手

AgentId：
- V2AgentId.EMAIL
- wireName：email

触发关键词/正则：
- 邮件
- 邮箱
- email
- smtp
- 发信
- 发送给
- 发给
- Regex：
  - (发|发送).*(邮件|email|邮箱)
  - (发给|发送给|发送至|发至|给) + 邮箱地址
  - 单独邮箱地址

优先级：
- priority = 8

特殊路由：
- explicitEmailSend 会将 “发送给/发给 + 邮箱地址” 提升到高置信度。
- 即使智能搜索开关开启，明确邮件发送请求也会优先进入 Email Agent。

依赖工具：
- ONE_LLM
- ROOM_MEMORY
- FLOW_STREAMING
- JAVA_MAIL_QQ_SMTP
- DOCUMENT_WRITER

执行器：
- V2RuntimeEmailExecutor

执行流程：
1. parseEmailRequest 解析：
   - to：邮箱地址
   - subject：主题；缺省为“来自 GeoAgent 的邮件”
   - content：正文
2. 正文解析规则：
   - 明确“内容：/内容为/内容是/正文：/正文为/正文是”时取后续文本。
   - 如果用户说“上述内容/上面内容/以上内容/这些内容/这些新闻/刚才的内容”，则取最近一条 assistant 消息作为邮件正文。
   - 否则默认使用用户输入作为正文。
3. gateway.sendEmail 调用 AuthRepository.sendEmail。
4. gateway.remember 写入 email 记忆。
5. 输出明确确认句：
   - 已经将xxx等内容发送给xxx@qq.com✅
6. 生成 email artifact：
   - to
   - subject
   - delivery

失败模式：
- 缺少收件人邮箱 -> NEEDS_INPUT
- SMTP/后端发送失败 -> BLOCKED

实际发送后端：
- V2ProductionRuntimeGateway.sendEmail
- 调用 authRepository.sendEmail(request.to, request.subject, request.content)


6. PDF Agent / PDF 解析

AgentId：
- V2AgentId.PDF
- wireName：pdf

触发关键词/正则：
- pdf
- PDF
- 提取pdf
- 解析pdf
- Regex：pdf|解析.*文件|提取.*文件

优先级：
- priority = 19

依赖工具：
- ONE_LLM
- ROOM_MEMORY
- FLOW_STREAMING
- PDF_PARSER
- FILE_ACCESS

执行器：
- V2RuntimePdfExecutor

执行流程：
1. gateway.listDocuments 获取本地文档列表。
2. 过滤 type == pdf 的文档。
3. selectDocumentForInput 根据输入匹配目标 PDF；如果没匹配则默认取第一个 PDF。
4. gateway.getDocumentText(pdf.id) 获取 PDF 文本。
5. gateway.streamWithMemory 调用 One LLM 基于 PDF 文本提取重点、结构和结论。
6. gateway.remember 写入 pdf 记忆。
7. 生成 pdf artifact：
   - document
   - name
   - chars

失败模式：
- 无 PDF 文档 -> NEEDS_INPUT
- 文档列表失败 -> BLOCKED
- PDF 文本读取失败 -> BLOCKED
- One LLM 合成失败 -> BLOCKED


7. File Agent / 文件管理

AgentId：
- V2AgentId.FILE
- wireName：file

触发关键词/正则：
- 文件
- 上传
- 打开文件
- 保存
- 导入
- Regex：
  - (文件|上传|导入|保存)
  - (读取|查看|打开|删除|移除).*文件扩展名

优先级：
- priority = 42

依赖工具：
- ONE_LLM
- ROOM_MEMORY
- FLOW_STREAMING
- FILE_ACCESS

执行器：
- V2RuntimeFileExecutor

执行流程：
1. gateway.listDocuments 获取本地文档。
2. selectDocumentForInput 按文档 id、完整文件名、去扩展名文件名匹配目标。
3. fileActionFor 判断动作：
   - 删除/移除/删掉/delete/remove -> DELETE
   - 读取/查看/打开/显示/内容/read/show/open -> READ
   - 其他 -> LIST

LIST 分支：
1. 输出本地文档列表。
2. 生成 file artifact：
   - documents
   - action = list
   - selectedId
   - selectedName
   - selectedType

READ 分支：
1. 要求 selected 不为空。
2. gateway.getDocumentText(selected.id)
3. 输出文档名和前 4000 字文本。
4. 生成 file artifact：
   - action = read
   - selectedId
   - selectedName
   - chars

DELETE 分支：
1. 要求 selected 不为空。
2. gateway.deleteDocument(selected.id)
3. 输出“已删除文档：xxx”
4. gateway.remember 记录删除操作。
5. 生成 file artifact：
   - action = delete
   - selectedId
   - selectedName

失败模式：
- 没有本地文档 -> NEEDS_INPUT
- 读取/删除但未指定目标 -> NEEDS_INPUT
- 文档读取、删除失败 -> BLOCKED


8. UnitConversionAgent / 单位换算

文件：
- app/src/main/java/com/geoagent/agent/UnitConversionAgent.kt

失败模式：

八、V2ProductionRuntimeGateway 对接能力

文件：app/src/main/java/com/geoagent/agent/v2/V2ProductionRuntimeGateway.kt

1. completeWithOneLlm / streamWithOneLlm
   - 使用 DeepSeek API Key
   - 调用 DeepSeekChatClient
   - streamWithOneLlm 将 ChatEvent.Content 透传给 onContent

2. search
   - 使用 Tavily API Key
   - 调用 SearchUseCase.searchRequiredContext
   - 转换为 V2RuntimeSearchResult

3. retrieveRag
   - 从 DocumentStore 获取 chunks
   - 优先使用 SiliconFlow embedding 做相似度检索
   - 无 embedding 或失败时降级 keyword ranking

4. listDocuments / getDocumentText / deleteDocument
   - 通过 DocumentStore 读取或删除本地文档

5. prepareCalendarEvent
   - 通过 V2CalendarContractAdapter 构造系统日历插入 Intent 信息

6. sendEmail
   - 调用 AuthRepository.sendEmail
   - 当前邮件发送走 QQ SMTP/后端邮件接口

7. recallMemory / remember
   - 通过 V2MemoryRepository 读写 Room memory

8. saveTask / listTasks
   - 通过 V2MemoryRepository 保存和查询任务


九、Artifact 约定

多数 V2 Runtime Executor 会返回 artifact，用于后续 UI action、调试或结构化追踪。

常见 artifact 类型：
- search
- research
- schedule
- task
- task_list
- file
- pdf
- email

当前格式：
- 部分使用 v2Artifact(type, map)
- 部分旧代码仍使用简单字符串，如 task:id=...;title=...

使用位置：
- V2OrchestrationResult.runs[].artifact
- ChatManager 将 trace 写入 assistant Message.agentResultJson
- V2SystemActionMapper 读取 calendar/reminder/schedule/meeting/travel 等 artifact 并转成 UI action


十、排除模式与本文 Agent 的边界

1. 快速模式边界
   - 快速模式主要是普通 chatRepository.streamChat。
   - 只有当路由未进入本地直达 Agent、未进入 V2 runtime、也未启用 RAG 专用模式时，才落入普通聊天。
   - 本文不展开 ChatRepositoryImpl、DeepSeek 普通聊天请求、思考模式等快速链路。

2. RAG 模式边界
   - RAG 模式包括 currentMode == ChatMode.RAG 和 V2AgentId.RAG。
   - 本文只在 Research Agent、PDF/File Agent 涉及本地文档时提到文档能力，不展开 RAG Agent 的完整实现。

3. Search Agent 与“智能搜索”按钮边界
   - V2 Search Agent 是 Agent runtime 的搜索能力。
   - UI 上“智能搜索”按钮可能让普通 chat stream 使用 web_search 增强。
   - 两者相关但不是完全同一条执行链。
